Requisitos específicos para este MVP inicial (solo MikroTik):

1. Usar el MCP Server oficial de MikroTik: https://github.com/jeff-nasseri/mikrotik-mcp (Docker, Python, expone tools para VLAN, firewall, DNS, WireGuard, interfaces, queues, etc.).

2. Estructura del proyecto (genera todo esto ahora):
   - /docker-compose.yml (incluye: mikrotik-mcp, orchestrator, redis, qdrant vector-db, postgres para estado)
   - /orchestrator/ (FastAPI + LangGraph o CrewAI)
   - /agents/ (MonitoringAgent, ConfigAgent, FaultAgent – cada uno como módulo independiente)
   - /skills/ (wrapper MCP client que descubre automáticamente las tools del MikroTik MCP)
   - /intent-manager/ (RAG con políticas en español)
   - /digital-twin/ (RouterOS emulado ligero con Docker)
   - /dev-agent/ (meta-agente que pueda modificar su propio código)

3. Requisitos técnicos obligatorios:
   - Todos los agentes se comunican vía MCP (JSON-RPC) con el servidor MikroTik.
   - El Orchestrator usa A2A (Agent-to-Agent) para delegar tareas.
   - Closed-loop ZSM style: Monitor → Detect anomaly → Plan → Execute via MCP → Verify.
   - Seguridad: API keys en .env, RBAC básico, aprobación humana opcional.
   - Idiomas: todo en español (intents y logs).
   - Documentación: README.md con pasos de instalación + ejemplo de intent.

4. Genera en orden:
   Paso 1: docker-compose.yml completo y listo para `docker compose up`
   Paso 2: Estructura completa de carpetas y archivos __init__.py
   Paso 3: Código del MCP client wrapper (mikrotik_skill.py) que liste y llame todas las tools disponibles
   Paso 4: Código mínimo del Super-Orchestrator (orchestrator.py) que acepte cualquier intent
   Paso 5: Un ejemplo completo de agente de configuración (ConfigAgent) que pueda crear VLAN + firewall
   Paso 6: README.md con comandos para lanzar y prueba inicial (“Optimiza el firewall para bloquear tráfico de una IP”)

Usa las mejores prácticas 2026: pydantic v2, async everywhere, logging estructurado, error handling con retry.
